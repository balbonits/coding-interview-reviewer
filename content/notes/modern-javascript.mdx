If your JavaScript is from the ES5 / "always use `var`" era, here's everything that's been added since — organized by what it solves, not by ECMAScript version. Every feature here ships in current Chrome, Safari, Firefox, Edge, Node, and Bun.

## Variables: stop using `var`

```js
let count = 0;          // block-scoped, can reassign
const PI = 3.14;        // block-scoped, cannot reassign
```

`var` is function-scoped and hoisted in ways that surprise everyone. `let` and `const` are block-scoped (`{ ... }`) and don't hoist past the block. Default to `const`; use `let` only when you need to reassign.

## Functions: arrow functions

```js
const add = (a, b) => a + b;
const square = x => x * x;
const log = () => console.log("hi");

// Implicit return for a single expression. Add { } for a body:
const greet = name => {
  const msg = `Hello, ${name}`;
  return msg;
};
```

Two reasons arrow functions matter:
1. **Concise syntax** for callbacks
2. **Lexical `this`** — they don't get their own `this`, so you can use them in callbacks without `.bind(this)` or `const self = this`

## Template literals (backtick strings)

```js
const name = "Maya";
const greeting = `Hello, ${name}!`;     // string interpolation
const html = `
  <div>
    Multi-line
    strings
  </div>
`;
```

## Destructuring

Pull values out of objects and arrays in one line.

```js
// Object
const { name, email } = user;
const { name: userName } = user;             // rename
const { name = "Anon" } = user;              // default if undefined

// Array
const [first, second, ...rest] = [1, 2, 3, 4, 5];

// In function parameters
function greet({ name, age = 18 }) { ... }
```

## Spread and rest

```js
// Spread (expand)
const merged = { ...a, ...b };           // merge objects
const copy   = [...arr];                 // shallow-copy array
fn(...args);                             // expand args

// Rest (collect)
function log(...args) { console.log(args); }     // collects into array
const { name, ...everythingElse } = obj;
```

## Default parameters

```js
function greet(name = "World") {
  return `Hello, ${name}`;
}
greet();        // "Hello, World"
greet("Maya");  // "Hello, Maya"
```

## Modules (import/export)

```js
// utils.js
export function add(a, b) { return a + b; }
export const PI = 3.14;
export default function main() { ... }    // one default per file

// app.js
import main, { add, PI } from './utils.js';
import * as utils from './utils.js';
```

ES modules replaced CommonJS (`require`) and AMD. Top-level `await` works inside modules:

```js
// in module.js — no async wrapper needed
const data = await fetch('/api/data').then(r => r.json());
```

## Classes

```js
class Animal {
  #species;                            // private field (real privacy, not _convention)

  constructor(species) {
    this.#species = species;
  }

  get species() { return this.#species; }

  static fromString(s) {               // static method
    return new Animal(s);
  }
}

class Dog extends Animal {
  bark() { return `${this.species} says woof`; }
}
```

Note `#privateField` — real private state, enforced by the runtime. Replaces the `_underscore` convention.

## Promises and async/await

```js
// The async/await form is the modern style:
async function loadUser(id) {
  try {
    const res = await fetch(`/api/users/${id}`);
    if (!res.ok) throw new Error(`Status ${res.status}`);
    return await res.json();
  } catch (err) {
    console.error(err);
    return null;
  }
}

// Concurrent waits:
const [user, posts] = await Promise.all([loadUser(1), loadPosts(1)]);

// Race / first-resolved:
const fastest = await Promise.race([fetch(a), fetch(b)]);

// Get all results, success or failure:
const settled = await Promise.allSettled([loadUser(1), loadUser(2)]);
// → [{ status: 'fulfilled', value: ... }, { status: 'rejected', reason: ... }]

// First success:
const firstOk = await Promise.any([fetch(a), fetch(b)]);
```

## Optional chaining `?.` (the null-checking life-saver)

```js
// Before:
const city = user && user.address && user.address.city;

// After:
const city = user?.address?.city;       // undefined if any link is null/undefined

// Works on function calls and array access:
user?.greet?.();                        // call only if greet exists
arr?.[0];                               // access only if arr exists
```

## Nullish coalescing `??`

```js
// `||` treats 0 and "" as falsy → bugs:
const port = config.port || 3000;       // 0 → 3000  ← wrong if user wanted 0!

// `??` only falls back on null or undefined:
const port = config.port ?? 3000;       // 0 → 0     ← correct
```

Combine with optional chaining:

```js
const name = user?.profile?.name ?? "Anonymous";
```

### Logical assignment

```js
opts.timeout ??= 5000;      // assign 5000 only if opts.timeout is null/undefined
opts.retry  ||= 3;          // assign 3 only if opts.retry is falsy
opts.debug  &&= !!flag;     // reassign debug only if it's already truthy
```

## structuredClone() — proper deep cloning

```js
const clone = structuredClone(original);
```

Built into the runtime. Handles `Map`, `Set`, `Date`, `ArrayBuffer`, `RegExp`, `Blob`, and **circular references**. The `JSON.parse(JSON.stringify(x))` hack is officially obsolete.

```js
const node = { id: 1 };
node.self = node;                     // circular!
JSON.stringify(node);                 // throws
structuredClone(node);                // works
```

## Iteration and collections

### Map / Set

```js
const counts = new Map();
counts.set('apples', 3);
counts.get('apples');               // 3
counts.size;                        // 1

const seen = new Set([1, 2, 2, 3]);
seen.has(2);                        // true
seen.size;                          // 3 (dedupes automatically)
```

`Map` keys can be any type (objects too); plain object keys are always strings/symbols.

### Set methods (ES2024)

```js
const a = new Set([1, 2, 3]);
const b = new Set([2, 3, 4]);

a.intersection(b);                  // Set { 2, 3 }
a.union(b);                         // Set { 1, 2, 3, 4 }
a.difference(b);                    // Set { 1 }
a.isSubsetOf(b);                    // false
```

Stop reaching for Lodash for set arithmetic.

## Array methods worth knowing

```js
// Existence checks
[1,2,3].includes(2);                // true (use this instead of indexOf !== -1)

// Find an element
[1,2,3].find(x => x > 1);           // 2
[1,2,3].findIndex(x => x > 1);      // 1
[1,2,3].findLast(x => x < 3);       // 2

// Negative indexing (ES2022)
[1,2,3].at(-1);                     // 3   (cleaner than arr[arr.length - 1])

// Flatten
[[1,2],[3,4]].flat();               // [1,2,3,4]
[1,2,3].flatMap(x => [x, x*2]);     // [1,2,2,4,3,6]

// Non-mutating versions (ES2023)
const sorted   = arr.toSorted();    // returns sorted copy; arr untouched
const reversed = arr.toReversed();
const spliced  = arr.toSpliced(1, 1);
const updated  = arr.with(0, 'new'); // index 0 replaced, returns copy
```

The `to*` family is huge for React state and immutable data — no more `[...arr].sort()` ceremony.

## Object utilities

```js
Object.keys(obj);                   // ['a', 'b', 'c']
Object.values(obj);                 // [1, 2, 3]
Object.entries(obj);                // [['a', 1], ['b', 2]]
Object.fromEntries([['a', 1]]);     // { a: 1 }   (inverse of entries)
Object.hasOwn(obj, 'foo');          // safer than obj.hasOwnProperty('foo')
```

`Object.fromEntries` is great for building objects from `Map`s, query strings, or `entries().filter(...)` chains.

## BigInt

```js
const big = 9007199254740993n;      // n suffix
typeof big;                         // 'bigint'
big + 1n;                           // 9007199254740994n
```

For integers beyond `Number.MAX_SAFE_INTEGER` (2⁵³-1). Don't mix BigInt with Number arithmetic.

## What you can finally drop

- `var` (always)
- `arguments` object (use `...rest`)
- `function() { var self = this; }` patterns (use arrow functions)
- jQuery for AJAX (`fetch`)
- `_underscore` private convention (use `#privateFields`)
- `JSON.parse(JSON.stringify(x))` for deep clone (use `structuredClone`)
- Lodash `_.get(obj, 'a.b.c')` (use `obj?.a?.b?.c`)
- Lodash `_.intersection`, `_.difference` for Sets (use Set methods)
- Lodash `_.sortBy` for non-mutation (use `arr.toSorted(cmp)`)
- IIFE module pattern (`(function() { ... })()`) — just use ES modules

## Quick reference

| Goal | Old way | Modern way |
|---|---|---|
| Block-scoped variable | `var` (function-scoped, surprising) | `let` / `const` |
| Concise callback | `function (x) { return x*2 }` | `x => x*2` |
| String interpolation | `"Hi " + name` | `` `Hi ${name}` `` |
| Default arg | `x = x \|\| 1` | `function f(x = 1)` |
| Deep clone | `JSON.parse(JSON.stringify(x))` | `structuredClone(x)` |
| Null-safe property access | `a && a.b && a.b.c` | `a?.b?.c` |
| Default for `null` only | `x \|\| 'd'` (unsafe for 0/"") | `x ?? 'd'` |
| Find element | `for` loop with `if` | `arr.find(p)` |
| Async control flow | nested callbacks / `.then` chains | `async/await` |
| Module system | CommonJS / AMD | ES modules (`import`/`export`) |
